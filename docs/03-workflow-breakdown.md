# Workflow Breakdown

Kourai is built around multi-step workflows rather than one-shot AI generation.

## Core Content Workflow

1. A campaign goal, topic, or account context starts the workflow.
2. A strategy step creates platform-specific directions.
3. A draft step turns strategy into content options.
4. An enhancement step improves tone, clarity, or platform fit.
5. A gatekeeper step reviews quality, risk, and brand fit.
6. Approved items remain human-controlled before scheduling or publishing.
7. Logs and performance records become inputs for future learning.

## Validated PHINN Bootstrap Workflow

At a public-safe level, the PHINN bootstrap workflow is:

1. Create or validate the database.
2. Apply pending migrations.
3. Create a backup.
4. Synchronize the social-account record.
5. Import or prepare owned historical records.
6. Calculate historical rankings.
7. Optionally enrich records with AI when configuration is available.
8. Generate or update voice-learning data.
9. Validate reports and stored results.
10. Remain in analyze-only mode with posting disabled.

## In Development

- Historical-post importing and sync behavior
- Voice and personality analysis
- Adaptive voice profiles
- Performance-based recommendations
- Account memory and strategy-history interfaces

The public repo describes these workflows at the system level only. Production scripts, API calls, scoring details, prompts, credentials, and raw account data stay private.
